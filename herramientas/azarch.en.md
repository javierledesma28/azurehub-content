## AzArch

AzArch is the reference center for official architecture patterns from Microsoft Architecture Center. Filter by workload, search by technology, and if you like a pattern, import it into the AzDraw canvas with one click — no more 20 minutes copying out of a PDF.

## What it's for

- Find the reference pattern for a case (3-tier web, multi-region AKS, event-driven, etc.)
- Browse ~200 indexed patterns without jumping across docs pages
- Filter by industry, technology, or WAF pillar
- Import the pattern as the starting point for your next diagram

## Getting started

1. **Step 1**: Open AzArch from the Hub.
2. **Step 2**: Use the search bar or filters (industry, included services, complexity).
3. **Step 3**: Click a pattern to see the diagram, description, and components. Click "Import to AzDraw" to start editing.

## Common use cases

- **Project kickoff**: you need the baseline "AKS production-grade" reference and don't want to invent it from scratch
- **Compare options**: API Management + Functions vs Front Door + App Service? See both patterns side by side
- **Teach the team**: the junior sees the official pattern before proposing one

## Tips

- Filter by "Well-Architected" if you want patterns that already passed formal review
- After importing to the canvas, validate with AzReview — the pattern is a starting point, not a final answer

## FAQ

**Are the patterns up to date?**
We sync against Microsoft Architecture Center; if Microsoft updates a pattern, we refresh it in the next release.

**Can I submit my own pattern?**
Not today. AzRecipes covers Hub-curated architectures.
