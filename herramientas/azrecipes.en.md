## AzRecipes

AzRecipes is the catalog of "recipes" — pre-built, Hub-curated architectures for common cases: 3-tier web, hub-and-spoke, AKS production-grade, event-driven, data platform. Click, import to the canvas, and you've got a baseline to start.

## What it's for

- Kick off a project with a production-ready baseline instead of a blank canvas
- Compare recipes for the same case (e.g. small AKS vs multi-region AKS)
- Import the full recipe to the AzDraw canvas
- Have a curated set of validated architectures

## Getting started

1. **Step 1**: Open AzRecipes and filter by category (web, data, AI, hybrid, etc.).
2. **Step 2**: Click a recipe to see the diagram, components, and notes.
3. **Step 3**: Import to AzDraw — start editing your version.

## Common use cases

- **Kickoff**: new project, you want a production AKS baseline without inventing it
- **Internal templates**: replicate the same base across a client's projects
- **Teaching**: the junior sees the full recipe before proposing their own

## Tips

- AzRecipes gives you the baseline; client-specific tuning is on you
- If your org has internal patterns, AzDraw lets you save diagrams as templates

## FAQ

**Difference vs AzArch?**
AzArch indexes official Microsoft patterns. AzRecipes are Hub-curated recipes (can combine multiple patterns + design opinions).

**Can I contribute recipes?**
Not via UI yet. If you have a good one, send a PR.
